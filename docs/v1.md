# QoL Config v1

`qol-config.toml` is the plugin-authored configuration contract for the QoL ecosystem.

## Goals

- one author-facing config file per plugin
- declarative data, not executable config
- stable validation in CI
- normalized form output for `qol-tray`

## Authoring Model

Each plugin may ship:

- `plugin.toml`
- `qol-config.toml`

`qol-tray` owns user overrides outside the plugin repo. Effective config is:

- defaults from `qol-config.toml`
- merged with tray-owned user overrides
- stale override keys are ignored
- invalid override values are rejected during normalization

## Top-Level Keys

- `schema_version`
- `title`
- `description`

## Tables

- `[section.<id>]`
- `[field.<id>]`

Section ids and field ids are unique within the file.
Section order and field order are preserved from author input.

## Sections

Supported section keys:

- `label`
- `description`
- `actions`

Example:

```toml
[section.snap]
label = "Snap"
actions = ["snap-left", "snap-right", "snap-bottom"]
```

## Fields

Every field requires:

- `type`

Supported common field keys:

- `config_key`
- `label`
- `description`
- `placeholder`
- `section`
- `default`
- `option_labels`
- `show_when`

Supported field types in v1:

- `boolean`
- `string`
- `number`
- `select`
- `string_array`
- `object_array`
- `object_map`

### Number Fields

Supported extra keys:

- `min`
- `max`
- `step`

### Select Fields

Required extra key:

- `options`

Optional extra key:

- `[field.<id>.option_labels]`

Example:

```toml
[field.center_mode]
type = "select"
options = ["pixels", "percent"]

[field.center_mode.option_labels]
pixels = "Fixed Size"
percent = "Relative Size"
```

### Conditional Visibility

Fields may declare:

- `[field.<id>.show_when]`

Supported keys:

- `field`
- `equals`

Example:

```toml
[field.center_width_px.show_when]
field = "center_mode"
equals = "pixels"
```

`show_when` references another field id and compares against that field's current value.

## Example

```toml
schema_version = 1
title = "Window Actions"
description = "Window sizing and movement behavior."

[section.center]
label = "Center"
description = "Default centered window size."
actions = ["center"]

[field.center_mode]
type = "select"
label = "Sizing Mode"
section = "center"
default = "interpolate"
options = ["fixed", "percent", "interpolate"]

[field.center_width_px]
type = "number"
label = "Width"
section = "center"
default = 1152
min = 200

[field.center_width_px.show_when]
field = "center_mode"
equals = "fixed"

[field.center_height_px]
type = "number"
label = "Height"
section = "center"
default = 892
min = 200

[field.excluded_apps]
type = "string_array"
label = "Excluded Apps"
default = []
placeholder = "Bundle ID or app id"
```

## Validation Rules

v1 should reject:

- unsupported `schema_version`
- unknown field types
- duplicated section or field ids
- invalid section action ids
- duplicated section action ids
- field `section` references to missing sections
- `select` fields without `options`
- invalid default type for the declared field type
- invalid override type for the declared field type
- select values not present in `options`
- numeric values outside `min` and `max`
- numeric constraints where `min > max`
- `step <= 0`
- empty `options`

## Normalized Output

The normalized form exposed to `qol-tray` should contain:

- title
- description
- ordered sections
- section actions
- ordered fields
- field id
- field type
- label
- description
- placeholder
- current value
- default value
- constraints
- options

`qol-tray` should render the normalized output rather than reinterpreting raw TOML directly.
